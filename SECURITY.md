# 🔐 الأمان والحدود

هذا المستودع **عامّ** ووثائقيّ بحت (لا كود تشغيليّ). لا تُبلَّغ هنا ثغرات منتجات لغة ص — تلك تذهب إلى مستودع الكود المعنيّ.

## ما يَعني «الأمان» هنا: تسرّب الحدود

الخطر الأساسيّ هو **تسرّب محتوى لا ينتمي إلى مستودع عامّ**:

- **أرقام ماليّة / ميزانيات / تسعير / نسب ربح** ← تُدار في قنوات المنظمة الخاصّة، لا هنا.
- **مواد سرّيّة / استراتيجيّة / خطط منتجات مغلقة** ← لا مكان لها في مستودع عامّ.
- **مفاتيح/رموز/كلمات مرور** ← لا مكان لها في أيّ مستودع؛ أبطِلها فورًا (rotate).

يفرض هذا تلقائيًّا [`scripts/check_boundaries.py`](./scripts/check_boundaries.py) عبر CI على كلّ PR ودفع.

## إن رأيت تسرّبًا في تاريخ المستودع

1. لا تفتح Issue عامًّا يكرّر السرّ.
2. راسِل فريق core على البريد الموثّق في المنظمة، أو افتح تنبيهًا خاصًّا عبر GitHub Security Advisories.
3. للمفاتيح/الرموز: **أبطِلها (rotate) فورًا** — افتراض أنّها مكشوفة بمجرّد دخولها سجلّ عامّ.

> راجع [GOVERNANCE.md](./GOVERNANCE.md) للحدّ الكامل بين المستودعات.
