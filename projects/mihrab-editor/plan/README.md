# لوحة قيادة خطط محراب

> هذا المجلّد ليس خطّة واحدة بل **حافظة خطط** تتراكم عبر عمر المنتج. كل جهد تخطيطيّ مستقلّ يعيش في مجلّد مرقَّم تحت [`خطط/`](./خطط/)، والمعرفة الدائمة العابرة للخطط في [`مرجعية/`](./مرجعية/). هذا الفصل يمنع تسطّح المجلّد ويحفظ الاستمرارية.

## 🧭 القرار المعماريّ الحاكم

VSCodium-مرقَّع (لا تفرّع عميق ولا تجميد) + قاعدة الطبقات الثلاث. التفصيل الكامل في [بطاقة المشروع](../README.md).

## 📚 المرجعيّة (عابرة للخطط)

وثائق دائمة لا تخصّ مرحلة بعينها، تُحيل إليها كل الخطط:

- 📄 [خطة-الاختبارات.md](./مرجعية/خطة-الاختبارات.md) — هرم الاختبار، الطبقات السبع، مصفوفة المنصّات، نظام البوّابات التصاعديّة.
- 📄 [ميزات-تتمايز-عن-vscode.md](./مرجعية/ميزات-تتمايز-عن-vscode.md) — 8 ميزات تمايز، خريطة الميزة←الطبقة←دَيْن الدمج، ووعي صريح بما لا نتمايز فيه.

## 🗂️ اصطلاح التنظيم

```
plan/
├── README.md            ← هذه اللوحة (فهرس الخطط + المرجعيّة)
├── مرجعية/              ← معرفة دائمة عابرة للخطط
└── خطط/
    └── NN-اسم/          ← خطّة مستقلّة (README + وثائق مراحلها)
```

- **خطّة جديدة**: أنشئ `خطط/NN-اسم/README.md` وأضف صفًّا في جدول الخطط أعلاه.
- **معرفة دائمة** (لا تخصّ مرحلة): تذهب إلى `مرجعية/` لا داخل خطّة.
