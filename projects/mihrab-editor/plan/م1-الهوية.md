# م1 — الهويّة (تفصيل تنفيذيّ)

> الهدف: تحويل ثنائيّ VSCodium النظيف (م0) إلى محرّر باسم **محراب** ظاهريًّا 100% — دون أيّ منطق نواة. كل تغيير هنا **بيانات وأصول**، الطبقتان 2 و3 فقط. هذه أرخص مرحلة من حيث دَيْن الدمج بعد م0، وأعلاها أثرًا في الإدراك.

## مبدأ العزل

العلامة تُحقَن بآليّتين متمايزتين بدَيْن دمج مختلف:
- **`product.json`** (الطبقة 2): بيانات بحتة، دَيْن دمج شبه صفر — VSCodium يصمّمه نقطة تخصيص.
- **رُقَع الأصول** (الطبقة 3): شعارات/أيقونات/شاشة ترحيب، دَيْن منخفض لكن غير صفر (تتبع مواضع ملفّات المنبع).

## أ) حقول `product.json` الدنيا (الطبقة 2)

| الحقل | القيمة | الأثر |
|---|---|---|
| `nameShort` / `nameLong` | محراب / Mihrab | اسم النوافذ والقوائم |
| `applicationName` | `mihrab` | اسم تنفيذيّ CLI |
| `dataFolderName` | `.mihrab` | مجلّد بيانات المستخدم (عزل عن VS Code/VSCodium) |
| `win32AppUserModelId` / `win32*` | معرّفات محراب | تسجيل Windows، شريط المهامّ، المُثبِّت |
| `darwinBundleIdentifier` | معرّف محراب | حزمة macOS (لاحقًا) |
| `urlProtocol` | `mihrab` | روابط `mihrab://` العميقة |
| `extensionsGallery` | نقاط Open VSX | سوق الإضافات الافتراضيّ |
| `reportIssueUrl` / روابط الدعم | قنوات ص | المساعدة والإبلاغ |

> **حدّ حوكمة**: لا روابط لمستودعات خاصّة ولا قنوات داخليّة في منتج عامّ. روابط الدعم عامّة فقط.

## ب) رُقَع الأصول (الطبقة 3)

- **الأيقونات**: `ico` (Windows)، `icns` (macOS)، `png` بأحجامها (Linux/الواجهة).
- **شاشة الترحيب/البداية** ونصوصها العربيّة.
- **الثيم البصريّ الافتراضيّ** للعلامة (إن رُغب).
- تُحقَن بنمط رُقَع VSCodium في مواضع الأصول، كل رُقعة ذرّيّة وموثّقة.

## ج) ما لا يدخل م1 إطلاقًا

- أيّ منطق وظيفيّ (LSP، تحرير، RTL) ⇒ مراحل لاحقة.
- أيّ تعديل في سلوك النواة ⇒ م3 حصرًا.
- أيّ سرّ أو إشارة لمنتج مغلق ⇒ ممنوع في بناء عامّ.

## بوّابة الخروج (DoD لـم1)

- المحرّر يُقلع باسم «محراب»، أيقوناته وشاشته ومجلّد بياناته كلّها محراب.
- سوق الإضافات يشير إلى Open VSX.
- لا أثر بصريّ أو نصّيّ لـVS Code/VSCodium في تجربة المستخدم.
- التزامن مع المنبع ما زال قائمًا (لم نلمس منطقًا).

## مخاطر م1

- **تسرّب علامة مايكروسوفت/VSCodium** في زاوية منسيّة (about، روابط) ⇒ خلل هويّة (وربما ترخيص). علاجه: قائمة تدقيق شاملة لكل ظهور اسم.
- **تعارض `dataFolderName`** مع تثبيت VS Code للمستخدم ⇒ إفساد إعدادات. علاجه: مجلّد بيانات معزول `.mihrab`.
- **زحف رُقَع الأصول** نحو تعديل تخطيط ⇒ تخطّي حدّ م1. علاجه: حصر م1 في الأصول البحتة، وأيّ تخطيط يُرحَّل لـم3.
