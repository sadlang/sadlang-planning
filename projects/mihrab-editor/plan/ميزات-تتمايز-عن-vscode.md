# ميزات محراب المتمايزة عن VS Code

> القاعدة: لا نُعيد بناء محرّر لننافس VS Code في ما يُتقنه، بل لنقدّم ما **يستحيل** عليه أو ما يهمله — تجربة لغة ص العربيّة من الدرجة الأولى. كل ميزة أدناه إمّا غير موجودة في VS Code، أو موجودة بصورة لا تكفي العربيّة-كشيفرة.

## 1. تخطيط واجهة RTL أصيل
- **الفجوة في VS Code**: لا يدعم تخطيط workbench من اليمين لليسار إطلاقًا (طلب قديم غير منجَز).
- **في محراب**: شريط النشاط، الجانبيّ، اللوحات، القوائم — كلّها RTL. هذا أبرز تمايز بصريّ ومبرّر بناء محرّر خاصّ.
- **الطبقة**: رُقعة نواة (م3).

## 2. العربيّة-كشيفرة من الدرجة الأولى (bidi)
- **الفجوة**: معالجة VS Code لمحارف bidi مضبوطة لكشف «Trojan Source» في لغات لاتينيّة، وهي عدائيّة لشيفرة كلماتها ومعرّفاتها عربيّة.
- **في محراب**: رسم bidi، حركة مؤشّر، تحديد، والتفاف أسطر مضبوطة للنصّ العربيّ كلغة برمجة لا كتعليق.
- **الطبقة**: رُقعة نواة (م3).

## 3. سلسلة أدوات ص بتهيئة صفريّة
- **الفجوة**: VS Code محايد للّغات؛ كل لغة تحتاج تثبيت إضافة + ضبط مسار مترجم/خادم يدويًّا.
- **في محراب**: المترجم (`sad-build`)، المشغّل (`sad-run`)، وخادم LSP **محزومة داخل المنتج**؛ تثبيت واحد ⇒ بيئة ص كاملة تعمل فورًا.
- **الطبقة**: حزم (م2).

## 4. تعريب كامل للواجهة والمعجم
- **الفجوة**: تعريب VS Code جزئيّ وغير RTL.
- **في محراب**: واجهة عربيّة افتراضيّة متّسقة مع RTL، ورسائل/أوصاف مشتقّة من مصدر حقيقة اللغة (المعجم العربيّ).
- **الطبقة**: حزم + إعدادات (م1/م2).

## 5. تحرير واعٍ بقواعد ص
- **الفجوة**: إعداد لغة عامّ في VS Code.
- **في محراب**: إغلاق تلقائيّ، إزاحة ذكيّة، وطيّ مضبوط لقواعد ص العربيّة، مبنيّ على نمط Strategy القابل للتوسّع (يبني على إضافة vscode-rtl).
- **الطبقة**: إضافة (طبقة 1).

## 6. أدوات SadUI مدمجة
- **الفجوة**: لا شيء في VS Code لنظام الرسومات الخاصّ بـص.
- **في محراب**: لوحات/معاينة لنظام SadUI مدمجة في المنتج، تتطوّر مع مشروع الرسومات.
- **الطبقة**: إضافة (طبقة 1)، تتوسّع لاحقًا.

## 7. سوق منسّق للغة ص
- **الفجوة**: سوق VS Code ملكيّ لمايكروسوفت وممنوع على التفرّعات.
- **في محراب**: Open VSX كسوق افتراضيّ، مع إمكان منظومة إضافات ص منسّقة.
- **الطبقة**: مساهَمة بناء (`product.json`، م1).

## 8. النجمة القطبيّة: أدوات مكتوبة بلغة ص (dogfooding)
- **الفجوة**: أدوات VS Code مكتوبة بـTypeScript حصرًا.
- **في محراب**: حين تنضج خلفيّة JS/WASM في مترجم ص، يُعاد كتابة خادم LSP (ثم أدوات أخرى) بلغة ص ويُستبدَل داخل المنتج — دليل حيّ على نضج اللغة. **متعدّد سنوات، لا يُلتزَم الآن.**

## ما لا نتمايز فيه (وعيٌ صريح)

لتجنّب توهّم القيمة: محراب **يرث** من VS Code (ولا يحاول التفوّق فيه): محرّك Monaco، نظام الإضافات، التصحيح، الطرفيّة، Git، الأداء. تمايزنا **حصرًا** في طبقة لغة ص العربيّة وRTL. كل جهد يُصرَف لإعادة اختراع ما يرثه المحرّر = هدر ودَيْن دمج بلا مقابل.

## خريطة الميزة ← الطبقة ← دَيْن الدمج

| الميزة | الطبقة | دَيْن الدمج |
|---|---|---|
| 3 تهيئة صفريّة · 5 تحرير ص · 6 SadUI · 8 dogfooding | إضافة | صفر |
| 4 تعريب · 7 سوق | مساهَمة بناء | منخفض |
| 1 workbench RTL · 2 bidi | رُقعة نواة | مرتفع (مبرَّر) |

> الخلاصة: **6 من 8 ميزات تمايز بدَيْن دمج صفر/منخفض.** فقط ميزتا RTL النواتيّتان تحملان دَيْنًا مرتفعًا — وهما بالضبط مبرّر وجود محراب. أي أنّ التمايز الأكبر يقع حيث الكلفة مبرّرة، والباقي شبه مجّانيّ.
