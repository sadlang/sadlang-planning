## 🎯 الهدف

توفير بيئة تطوير متكاملة (IDE) للغة ص باسم **محراب**، تعمل بتهيئة صفريّة، وتقدّم تجربة RTL لا يضاهيها VS Code الأصليّ، دون أسر فريق الهندسة في دورة إصدارات VS Code.

## 🧭 القرار المعماريّ الحاكم

اختير **VSCodium-المرقَّع** على التفرّع العميق (نمط Cursor) وعلى التجميد (hard fork)، للأسباب:

- **التجميد لا يلغي دَيْن الصيانة بل يبدّله**: يستبدل دَيْن دمج VS Code (الذي تتحمّل مايكروسوفت ثقله الأمنيّ ودعم Chromium ونظم التشغيل) بدَيْن أمنيّ وشيخوخيّ يتحمّله الفريق وحده — وهو أثقل.
- **ملكيّة الهويّة ≠ تباعد قاعدة الشيفرة**: VSCodium محرّر خاصّ 100% (اسم، علامة، سوق Open VSX) ومع ذلك يبقى متزامنًا مع المنبع تلقائيًّا.
- **الرافعة الحقيقيّة هي موضع التغييرات لا حقيقة التفرّع**: التغييرات الطبقيّة الإضافيّة تُبقي دَيْن الدمج شبه صفريّ.

### قاعدة الطبقات الثلاث (مرتّبة من الأرخص صيانةً)

1. **إضافة محزومة** — يُوضع هنا ~90% من منطق ص.
2. **مساهَمة بناء** (`product.json`، إعدادات افتراضية، أدوات مدمجة) — بلا شيفرة نواة.
3. **رُقعة وقت-البناء** (بنمط رُقَع VSCodium) — فقط لما يستحيل في 1 و2.

> القاعدة الذهبيّة: كلّما صعد التغيير طبقةً ارتفع دَيْن الدمج. لا تصعد إلّا مضطرًّا.
