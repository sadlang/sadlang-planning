# 🤝 المساهمة في sadlang-planning

## إضافة خطة مشروع جديد

1. انسخ القالب:
   ```bash
   cp -r projects/_TEMPLATE projects/<اسم-المشروع>
   ```
2. عدّل `projects/<اسم-المشروع>/README.md` (بطاقة المشروع: الحالة، المالك، المعالم، رابط مستودع الكود).
3. املأ `plan/` بخطة التنفيذ والمراحل والمهام.
4. أضف تقارير التقدّم في `reports/` (تسمية مقترحة: `YYYY-MM-DD-عنوان.md`).
5. ضع المخططات في `diagrams/` (يُفضَّل Mermaid، وتُقبل الصور وملفات PDF).
6. افتح Pull Request.

## تحديث خطة قائمة

- حدّث بطاقة المشروع وحالتها.
- أضف تقرير تقدّم جديد بدل تعديل القديم (للحفاظ على السجلّ التاريخيّ).

## قبل فتح PR — تحقّق من الحدود

شغّل المدقّق محلّيًّا لتفادي فشل CI:

```bash
python scripts/check_boundaries.py
```

يرفض المدقّق: الأرقام الماليّة، الأسرار الاستراتيجيّة، خطط المنتجات المغلقة. راجع [GOVERNANCE.md](./GOVERNANCE.md) للحدود الكاملة.

## أسلوب الكتابة

- العربيّة هي اللغة الأساسيّة، اتجاه RTL.
- التواريخ مطلقة (`2026-06-28`) لا نسبيّة.
- كلّ خطة تشير إلى مستودع الكود برابط، ولا تكرّره.
