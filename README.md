# 📋 sadlang-planning — خطط مشاريع ص المفتوحة المصدر

> المستودع العامّ المسؤول عن **خطط التنفيذ التشغيليّة** لكلّ مشروع مفتوح المصدر في منظومة لغة **ص (Sad Lang)**.

هذا المكان الذي يجتمع فيه **كيف ننفّذ** كلّ مشروع مفتوح: المراحل، المعالم، المهام، تقارير التقدّم، المخططات، والوثائق — في مجلد كامل لكلّ مشروع، **دون لمس مستودعات الكود**.

---

## ⭐ لماذا يوجد هذا المستودع؟

- ✅ **يجمع كلّ خطط المشاريع المفتوحة في مكان واحد** دون الحاجة للدخول إلى مستودعات الكود.
- ✅ **يمنح كلّ مشروع مجلدًا كاملًا** (لا ملفًّا واحدًا): خطة، تقارير، مخططات، أصول.
- ✅ **يستوعب التقارير والوثائق والجداول والمخططات وملفات PDF وملفات التحليل** دون أن تتلوّث مستودعات الكود.
- ✅ **يتيح لمدراء التنفيذ والمنتجات العمل دون لمس الكود** — صلاحيات مستقلّة على التخطيط وحده.

---

## 🗂️ البنية

```
sadlang-planning/
├── README.md            ← أنت هنا
├── GOVERNANCE.md        ← الحدّ الفاصل عن roadmap / internal / finance (اقرأه أولًا)
├── CONTRIBUTING.md      ← كيف تضيف/تحدّث خطة مشروع
├── SECURITY.md          ← الإبلاغ عن تسرّب الحدود وإبطال المفاتيح
├── CODEOWNERS           ← مالكو التخطيط (planning) + إشراف core
├── LICENSE              ← CC BY 4.0 (محتوى وثائقيّ لا كود)
├── projects/
│   ├── README.md        ← فهرس كلّ المشاريع وحالتها
│   ├── _TEMPLATE/       ← انسخ هذا المجلد لبدء خطة مشروع جديد
│   │   ├── README.md    ← بطاقة المشروع (الحالة، المالك، المعالم)
│   │   ├── plan/        ← خطة التنفيذ والمراحل والمهام
│   │   ├── reports/     ← تقارير التقدّم الدوريّة
│   │   ├── diagrams/    ← مخططات (Mermaid / صور / PDF)
│   │   └── assets/      ← أصول داعمة (جداول، تحليلات، PDF)
│   └── <اسم-المشروع>/   ← مجلد كامل لكلّ مشروع مفتوح
├── scripts/
│   └── check_boundaries.py   ← مدقّق الحدود (يمنع تسرّب الأرقام الماليّة/الأسرار)
└── .github/workflows/
    └── boundaries.yml        ← يشغّل المدقّق على كلّ PR
```

## 🚦 ما يدخل هنا وما لا يدخل

| يدخل ✅ | لا يدخل ❌ |
|---|---|
| خطط تنفيذ المشاريع المفتوحة | كود المصدر (← مستودعات الكود) |
| تقارير تقدّم ومعالم | أرقام ماليّة/ميزانيات |
| مخططات تشغيليّة وجداول | مواد سرّيّة أو استراتيجيّة |
| وثائق تحليل وPDF عامّة | الحوكمة الفيدراليّة (← `sadlang-roadmap`) |

> ⚠️ هذا المستودع **عامّ**. أيّ رقم ماليّ أو خطة منتج **مغلق المصدر** أو أسرار استراتيجيّة لا مكان لها هنا — يفرض ذلك مدقّق الحدود آليًّا. راجع [GOVERNANCE.md](./GOVERNANCE.md).

---

## 🚀 البدء بخطة مشروع جديد

```bash
cp -r projects/_TEMPLATE projects/<اسم-المشروع>
# عدّل projects/<اسم-المشروع>/README.md ثم املأ plan/ و reports/
```

ثمّ افتح Pull Request. سيمرّ مدقّق الحدود تلقائيًّا.
